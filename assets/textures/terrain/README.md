# Terrain Textures — Placeholder Assets

> **WARNING: These are placeholder textures generated programmatically.**
> They are mathematically correct KTX2 files but are deliberately ugly
> solid-colour images. They exist solely to allow the splatmap shader to
> be validated without waiting on real art.

## Layout

```
assets/textures/terrain/
  rock/
    albedo.ktx2     — mid-grey 50% RGB (128, 128, 128)
    normal.ktx2     — flat normal (128, 128, 255)
    roughness.ktx2  — 70% grey (179, 179, 179)
  grass/
    albedo.ktx2     — 30%R / 60%G / 20%B (77, 153, 51)
    normal.ktx2     — flat normal (128, 128, 255)
    roughness.ktx2  — 70% grey (179, 179, 179)
  sand/

    normal.ktx2     — flat normal (128, 128, 255)
    roughness.ktx2  — 70% grey (179, 179, 179)
  snow/
    albedo.ktx2     — 95% white (242, 242, 242)
    normal.ktx2     — flat normal (128, 128, 255)
    roughness.ktx2  — 70% grey (179, 179, 179)
```

## Specification

- Resolution: 256 × 256 pixels
- Format: VK_FORMAT_R8G8B8A8_UNORM, KTX2 container, no mipmaps, no supercompression
- Total bundle: ~3.1 MB (12 files × ~262 KB each)

## Replacement

These files should be replaced by a real art pass before the first content gate.
The splatmap shader (`crates/firewolf-client/src/terrain_material.rs`) expects
exactly these paths and the same KTX2 format. Mipmaps and BC7 compression are
welcome in the art-pass replacement.
